   목적에 따라 보다 효율적인 방법을 선택한다.

1. 그림의 디지털 표현 (참고 영상: https://youtu.be/99fk2wBrPgM )

  가. 비트맵(bitmap) 방식과 벡터 방식

구분

비트맵(bitmap, =raster image)

벡터(vector)

이미지

표현 방식

픽셀(pixel, 화소)이라는 아주 작은 사각형 색점들로 이미지를 표현하는 방식

점, 선, 면, 색상에 대한 정보를 수학식과 연산으로 표현하는 방식

예

특징

- 사진과 같이 정교한 이미지 표현 가능

- 벡터 방식에 비해 상대적으로 용량이 크고
 처리 속도 느림

- 확대, 축소 시 이미지 품질 저하됨

- 사진보다는 비교적 단순한 형태의 로고,
 픽토그램, 일러스트 등에 적합

- 비트맵 방식에 비해 상대적으로 용량이
 작고 처리 속도 빠름
- 확대, 축소해도 품질 변화 없음

파일 형식

- bmp: 압축을 하지 않거나 비손실 압축
           방식인 RLE 지원. 1~24비트 칼라.

- jpg: 압축율이 높고 손실이 적어 많이 사용

- gif: 8비트 칼라, 투명, 애니메이션 지원

- png: 압축율 높고 손실 적으며 투명 지원

- ai: 어도비사의 일러스트레이터 파일 형식

- svg: xml 기반의 공개 표준 벡터 파일 형식

  나. 해상도

    ① 개념1: 디지털 화면이나 이미지 파일을 구성하는 전체 픽셀의 개수
       예1) 모니터 해상도가 1366×768이다.

        예2) 컴퓨터 배경화면용 고해상도 이미지를 찾고 있다. ( 참고 자료: www.pixabay.com )

    ② 개념2: 디지털 이미지나 인쇄물에서 1인치 당 픽셀수(ppi, dpi)

        예) 웹용 이미지는 72ppi 이상, 인쇄용 이미지는 300dpi 이상이 적절하다.

웹 이미지 해상도 단위 PPI(pixels per inch)

인쇄물 해상도 단위 DPI(dots per inch)

2. 소리의 디지털 표현

  가. 웨이브 방식

    ① 아날로그 소리 파형을 디지털로 변환(pulse code modulation): 표본화 ➔ 양자화 ➔ 부호화

    ② 원본에 가까운 음질을 얻으려면?

      - 비트 수(bit depth)를 늘림: PCM 그래프에서 세로축 단위를 잘게 쪼개는 효과

      - 샘플링비율(sample rate)를 높임: PCM 그래프에서 가로축 단위를 잘게 쪼개는 효과

        단위) 1Hz(헤르쯔) = 초당 샘플링 수

      - 비트전송율(bit rate)를 높임: 초당 처리하는 비트 양을 늘리는 효과

        단위) 1bps(bits per second) = 초당 전송 비트수

      ⇒ 어느 정도 이상이 되면 더 이상 사람이 음질의 차이를 구별하지 못하므로 위의 요소들을
        계속 높이는 것은 의미가 없음

    ③ mp3 압축 방식: 사람이 들을 수 없는 영역의 주파수를 제거하는 손실 압축 방식

  나. 미디 방식 ( 참고 영상: https://youtu.be/sQgc5RcZn_g )

    ① 전자 악기끼리 디지털 신호를 주고받기 위해 신호를 규칙화한 것으로 음의 높낮이, 강약,
       길이, 시간, 박자 등을 코드화한 다음 코드를 조합하여 소리 표현(전자음)
